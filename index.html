The Problem Worth Solving

You Did the Hard Work. Canvas Is Still Empty.

Backward Design produces a rich artifact — outcomes, assessment evidence, a sequenced learning experience. That document represents real intellectual work. And then you open Canvas and face a blank page.

The design is done. The production hasn't started. Every module overview, assignment description, rubric, and student communication still has to be written — from scratch, in the RCE editor, one page at a time.

1

The Translation Gap

Design ≠ Delivery

A Course Design Document and a Canvas site are two different things. Converting one into the other is hours of formatting, writing, and copy-pasting — none of which improves the pedagogy.

2

The Time Tax

Every page. Every term.

Assignment descriptions. Rubric tables. Module overviews. Student announcements. Faculty write all of it — most of it from memory, most of it from scratch, most of it under deadline pressure.

3

The Missed Opportunity

The context is already there

Your outcomes, assessments, and module sequence are exactly what AI needs to produce usable first drafts. The Course Design Artifact is the prompt — it just hasn't been used that way yet.

The shift: Stop treating the Course Design Document as the finish line. It's the starting gun for an AI-assisted build that turns hours of production into minutes.

The Repeatable Workflow

5 Steps. Any Course. Every Time.

Start here each time you build or update a course. AI accelerates every step — but you direct each one.

1
Start with your Course Design Artifact

Your Backward Design work — outcomes, assessments, sequence — is the context AI needs. The richer the artifact, the stronger the first draft.

2
Generate a module structure

Paste objectives into the Module Builder prompt. Get a full week-by-week skeleton in one pass.

3
Expand each module → assignments + rubrics

One prompt per module. Output: instructions, rubric criteria, and a grading scheme table.

4
Draft student communications from real scenarios

Describe the situation. AI drafts; you approve. Covers announcements, reminder emails, and feedback templates.

5
Paste into Canvas → format → publish

Copy AI output directly into Canvas RCE. Apply DesignPLUS formatting for structure and accessibility.

Prompt Craft

The CRISP Framework

Bad AI output is almost always a bad brief. Five elements separate a usable first draft from a frustrating hallucination.

C Context Course name, student level, delivery modality, known pain points
R Role Tell AI who it is: "You are an instructional designer for a graduate business analytics course."
I Instruction The specific task: write, generate, draft, reformat, summarize
S Specs Format, tone, length, accessibility constraints, platform (Canvas RCE)
P Proof A standard or example to match: paste an existing assignment, rubric row, or email
Rule of thumb: If your prompt fits in one sentence, it probably needs more context. If it takes 10 minutes to write, that's the point — AI saves you the next hour.
Ready-to-Use Prompts

Prompt Library

Copy, fill in the highlighted fields, paste into Claude or ChatGPT. These follow the CRISP framework.

01 — Module Structure Builder
You are an instructional designer for [COURSE NAME], a [undergraduate / graduate] course delivered [online / hybrid / in-person]. Students struggle with [known pain point, e.g., "connecting theory to practice"]. Generate a module overview for Week [#]: [TOPIC]. Include: - 3 learning objectives (action-verb + measurable outcome) - A numbered sequence of activities (readings → practice → assessment) - An "estimated time" for each activity - One optional enrichment resource Format for Canvas LMS. Use plain language. Tone: [professional / conversational / encouraging].
02 — Assignment + Rubric Generator
You are an instructional designer for [COURSE NAME]. Write a complete assignment description for: [ASSIGNMENT TITLE] Learning objectives this assignment addresses: - [Objective 1] - [Objective 2] Include: - Purpose statement (2 sentences, student-facing) - Task instructions (numbered steps) - Deliverable specifications (format, length, submission type) - A 4-point analytic rubric with 3 criteria Rubric format: criteria | Excellent (4) | Proficient (3) | Developing (2) | Incomplete (1) Output in plain text formatted for Canvas RCE paste.
03 — Student Communication
You are a faculty member teaching [COURSE NAME]. Situation: [describe the scenario, e.g., "Several students missed the first quiz and haven't reached out. It's Week 3."] Write a Canvas Announcement that: - Opens with empathy, not a lecture - States one clear action the student should take - Includes your contact method and availability - Closes with encouragement - Tone: [warm / direct / formal] - Length: under 150 words Do not use the phrase "I hope this email finds you well."
04 — Module 1 Page Builder (See It)
You are an instructional designer for BUS 3410: Data Storytelling for Business, a 7-week accelerated undergraduate course delivered online. Module 1 covers Weeks 1–2. Title: "See It: Foundations of Data Visualization." Students struggle with selecting the right chart for their data and often default to bar charts for everything. Generate a complete Canvas module overview page including: - A 2-sentence module purpose statement (student-facing, energetic tone) - 3 measurable learning objectives using Bloom's action verbs - A sequenced activity list: Week 1: Reading + ungraded practice quiz Week 2: Graded assignment (Visualization Critique, 100 pts) - Estimated time for each activity - One "Pro Tip" callout with a practical insight students can apply immediately - Due date placeholder: [INSERT WEEK 2 DATE] Format for Canvas RCE. Use plain language. Tone: direct and encouraging. Do not use the phrase "In this module, you will."
05 — Module 2 Page Builder (Say It)
You are an instructional designer for BUS 3410: Data Storytelling for Business, a 7-week accelerated undergraduate course delivered online. Module 2 covers Weeks 3–5. Title: "Say It: Building the Data Narrative." Students often produce technically accurate visualizations but struggle to connect data findings to a clear recommendation. They treat charts as conclusions rather than evidence. Generate a complete Canvas module overview page including: - A 2-sentence module purpose statement (student-facing) - 3 measurable learning objectives - A sequenced activity list: Week 3: SCR Framework lecture (22 min) + reading Week 4: Storyboard draft + peer review discussion Week 5: Graded assignment (Narrative Memo, 150 pts) - Estimated time for each activity - One "Watch Out" callout warning students about the most common mistake in this module - Due date placeholders: [PEER REVIEW DUE DATE] and [MEMO DUE DATE] Format for Canvas RCE. Tone: direct and professional.
06 — Module 3 Page Builder (Show It)
You are an instructional designer for BUS 3410: Data Storytelling for Business, a 7-week accelerated undergraduate course delivered online. Module 3 covers Weeks 6–7. Title: "Show It: Presenting to Decision Makers." This is the capstone module. Students deliver a 10-minute live data presentation to a simulated executive panel. The most common failure point: students read from their slides instead of speaking to the data. Generate a complete Canvas module overview page including: - A 2-sentence module purpose statement that conveys the stakes and the opportunity - 3 measurable learning objectives - A sequenced activity list: Week 6: Dry run (recorded, ungraded) + Peer Q&A Feedback Forum Week 7: Live Executive Presentation (200 pts) - Estimated time for each activity - One "What Executives Notice" callout with 3 specific behaviors that distinguish strong presenters - Self-assessment checklist (5 items) students complete before submitting their final recording - Due date placeholders: [DRY RUN DATE] and [FINAL PRESENTATION DATE] Format for Canvas RCE. Tone: confident and motivating. This is the finish line.
